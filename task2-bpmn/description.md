# Описание процесса «Обработка и доставка заказа клиента»

## 1. Название процесса
Обработка и доставка заказа клиента

## 2. Владелец процесса
Яндекс Маркет

## 3. Участники процесса
- **Клиент** - инициирует заказ, оплачивает, получает товар
- **IT-система Яндекс Маркета** - принимает заказ, проверяет оплату, обрабатывает
- **Склад** - собирает и упаковывает товар
- **Отдел доставки** - назначает курьера и доставляет заказ

## 4. Описание процесса
Процесс начинается с того, что клиент размещает заказ на платформе Яндекс Маркет. IT-система принимает заказ и проверяет подтверждение оплаты через XOR-шлюз. Если оплата подтверждена, заказ передаётся на обработку (подпроцесс), где параллельно резервируется товар и формируются документы (AND-шлюз). Затем заказ передаётся на склад для сборки (подпроцесс), где товар находится, упаковывается и передаётся в отдел доставки. Курьер доставляет заказ клиенту.

## 5. Используемые элементы BPMN
- **Пулы:** 2 (Яндекс Маркет, Клиент)
- **Дорожки:** 3 (IT-система, Склад, Отдел доставки)
- **Шлюзы:** XOR (проверка оплаты), AND (параллельная обработка)
- **Подпроцессы:** 2 (Обработать заказ, Собрать заказ)
- **События:** Start, End, Intermediate Timer
- **Message Flow:** 2 (заказ клиента, товар доставлен)

## 6. Обоснование выбора элементов
- **XOR-шлюз** применён для проверки оплаты, так как возможны только два исхода: оплата подтверждена или нет.
- **AND-шлюз** использован для параллельного резервирования товара и формирования документов, что ускоряет обработку.
- **Collapsed Sub-Process** применены для упрощения главной диаграммы и детализации сложных участков на отдельных листах.
- **Intermediate Timer Event** в отделе доставки моделирует ожидание подтверждения от курьера.
- **Message Flow** обеспечивает корректное взаимодействие между пулами «Клиент» и «Яндекс Маркет».

## 7. Проверка по чек-листу
1. ✅ У процесса есть Start Event и End Event на каждой завершающей ветке
2. ✅ Sequence Flow не пересекает границу пула
3. ✅ Все исходящие потоки из XOR подписаны условиями
4. ✅ Имена задач в глагольной форме
5. ✅ Нет «висящих» стрелок
6. ✅ Подпроцессы раскрыты на отдельных листах
7. ✅ Дорожки соответствуют реальным ролям
8. ✅ Message Flow только между пулами
9. ✅ В каждом пуле свой Start Event
10. ✅ Каждая ветка заканчивается End Event с подписью
